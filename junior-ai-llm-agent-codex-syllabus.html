课程设计原则

这套课保留大学路线，但把公式、论文和复杂工程替换为演示、模仿、小工具和复盘。

概念保留

保留 LLM（Large Language Model，大语言模型）、RAG（Retrieval-Augmented Generation，检索增强生成/先检索资料再生成答案）、Agent（Agent，智能体/能计划并使用工具完成任务的 AI 系统）等概念名，每次出现都带括号解释。

实践驱动

每节课都用 Codex（Codex，AI 编程助手）完成一个可见结果：提示词卡、单词卡、错题卡、资料问答器、作息规划器或总项目页面。

Codex 贯穿

所有 demo 都基于 Codex（Codex，能写代码、读项目、调试和运行检查的 AI 编程助手），重点训练目标、上下文、限制、完成标准、验证和迭代。

每节课固定结构

时间环节说明
0-10 分钟理论引入讲 1-2 个核心概念，每个概念有括号解释

16 节课大纲

前 12 节建立概念和小工具能力，后 4 节完成基于 Codex（Codex，AI 编程助手）的学习生活小工具合集。

AI 不是魔法

用 Codex（Codex，AI 编程助手）把学习生活问题拆成计划，理解 AI（Artificial Intelligence，人工智能）、Search（Search，搜索）、Classification（Classification，分类）、Optimization（Optimization，优化）。产出：AI 使用安全约定卡。

安全与问题拆解

LLM 如何“接话”

用 Codex（Codex，AI 编程助手）做接话小游戏，理解 LLM（Large Language Model，大语言模型）、Token（Token，模型处理文字的小片段）、Training（Training，训练）、Inference（Inference，推理/生成）。产出：LLM 解释卡。

模型直觉

Prompt 与 Context

在 Codex（Codex，AI 编程助手）里对比好坏 Prompt（Prompt，提示词），学习目标、背景、限制、完成标准。产出：学生版 Prompt 模板。

表达训练

Hallucination 与 Evaluation

用 Codex（Codex，AI 编程助手）生成可检查的回答，训练检查 AI（Artificial Intelligence，人工智能）输出的方法。产出：AI 回答检查表。

判断力

Codex 与 Vibe Coding

认识 HTML（HyperText Markup Language，网页结构语言）和 Codex（Codex，AI 编程助手）。产出：个人兴趣主页。

第一个网页

CSS 与 UI

用 Codex（Codex，AI 编程助手）迭代页面风格，学习颜色、布局、字号、间距和 Iteration（Iteration，迭代）。产出：个人主页第二版。

界面表达

JavaScript 与单词翻牌卡

理解 Event（Event，事件）和 State（State，状态）。产出：英语单词翻牌卡。

交互入门

Debug 与 Test

学习 Reproduce（Reproduce，复现）和 bug 报告（错误报告）。产出：修复版单词卡。

验证习惯

RAG 与资料问答

用 Codex（Codex，AI 编程助手）制作基于资料回答的问答卡，理解 Retrieval（Retrieval，检索）和 Grounding（Grounding，基于资料回答）。产出：资料问答器原型。

资料能力

Embedding 与相似内容

用 Codex（Codex，AI 编程助手）设计错题分类标签，理解 Similarity（Similarity，相似度）和 Vector Database（Vector Database，向量数据库）。产出：错题分类器设计稿。

分类与检索

Tool Use 与文件协作

理解 File System（File System，文件系统）和 Browser Preview（Browser Preview，浏览器预览）。产出：小工具合集项目文件夹。

项目设计

项目集成

展示复盘

总项目：学习生活小工具合集

参考资料

大学课程参考

  • Stanford CS324: Large Language Models：斯坦福大学《大语言模型》课程。
    这门课系统讨论语言模型的能力、局限、风险、法律和社会影响。本课程借鉴它的整体视角，把第 1-4 课设计成 LLM（Large Language Model，大语言模型）基础直觉、Hallucination（Hallucination，幻觉）和 Evaluation（Evaluation，评估）意识。
  • Stanford CS25: Transformers United：斯坦福大学《Transformer 专题》课程。
    这门课围绕 Transformer（Transformer，现代大语言模型常用的神经网络结构）在语言、视觉、代码等领域的应用展开。本课程借鉴它对 Token（Token，模型处理文字的小片段）、上下文和模型结构直觉的重视。
  • UC Berkeley Large Language Model Agents：加州大学伯克利分校《大语言模型智能体》课程。
    这门课从 LLM（Large Language Model，大语言模型）能力出发，继续讨论 Agent（Agent，智能体）、工具使用、代码生成、网页自动化和复杂任务执行。本课程借鉴它的路线，把 Codex 放进计划、行动、验证、复盘的 Agent 思维中。
  • UC Berkeley Understanding Large Language Models：加州大学伯克利分校《理解大语言模型》课程。
    这门课关注 LLM（Large Language Model，大语言模型）的上下文、能力边界、训练方法和评估。本课程借鉴它的概念顺序，把 Prompt（Prompt，提示词）、Context（Context，上下文）和 Evaluation（Evaluation，评估）设计成反复练习的表达和判断能力。
  • UC Berkeley Agentic AI：加州大学伯克利分校《智能体式人工智能》课程。
    这门课关注 Agentic AI（Agentic Artificial Intelligence，具备行动和工具使用能力的人工智能）如何规划、调用工具和完成任务。本课程借鉴它的行动型 AI 视角，让学生通过 Codex demo 理解 Agent 不是只聊天，而是能分步骤做事。
  • CMU Large Language Models：卡内基梅隆大学《大语言模型》课程。
    这门课覆盖模型基础、应用、训练、评估和可靠性。本课程借鉴它的完整技术版图，但把复杂数学和工程实现降维成 RAG（Retrieval-Augmented Generation，检索增强生成）、资料问答、测试清单和学习小工具。
  • MIT 6.S191 Introduction to Deep Learning：麻省理工学院《深度学习导论》课程。
    这门课强调短时间讲清核心概念，并配合实验实践。本课程借鉴它的节奏，把每节 1 小时拆成理论引入、Codex demo、学生模仿、验证和复盘。
  • Harvard CS50 AI with Python：哈佛大学《基于 Python 的人工智能》课程。
    这门课把搜索、知识、推理、机器学习等 AI（Artificial Intelligence，人工智能）概念放进可完成的编程项目。本课程借鉴它的项目制思路，让学生每节课都有可见产出，并最终完成学习生活小工具合集。
  • Harvard SEAS Large Language Models：哈佛大学工程与应用科学学院《大语言模型》课程。
    这类课程通常同时覆盖 LLM（Large Language Model，大语言模型）的技术原理、应用场景和社会影响。本课程借鉴这种平衡，把能做什么和哪里要小心一起教。
  • Princeton COS 597G Understanding Large Language Models：普林斯顿大学《理解大语言模型》课程。
    这门课强调阅读、讨论、展示和对模型行为的深入理解。本课程借鉴它的表达与复盘方式，把第 16 课设计成 Demo（Demo，演示）、Reflection（Reflection，复盘）和下一步改进。
  • Johns Hopkins LLM Theory and Practice：约翰斯·霍普金斯大学《大语言模型：理论与实践》课程。
    这门课的定位是把理论理解和实践应用结合起来。本课程借鉴它的定位，保证学生不是只学会叫 AI 干活，也能理解 Token（Token，模型处理文字的小片段）、Inference（Inference，推理/生成）、RAG（Retrieval-Augmented Generation，检索增强生成）等基础概念。
  • University of Waterloo Foundations of LLMs：滑铁卢大学《大语言模型基础》课程。
    这门课强调 LLM（Large Language Model，大语言模型）的基础概念和应用理解。本课程借鉴它的基础路线，把大学里的专业内容压缩成适合初一学生的直觉。
  • ETH Zurich / Rycolab Large Language Models：苏黎世联邦理工学院 / Rycolab《大语言模型》课程。
    这门课系统覆盖语言模型的训练、推理、对齐和评估。本课程借鉴它的结构，把 Training（Training，训练）、Inference（Inference，推理/生成）和 Evaluation（Evaluation，评估）保留下来，但用类比和小工具实践解释。
  • University of Washington Hands-on Master Class on LLMs and ChatGPT：华盛顿大学《LLM 与 ChatGPT 动手大师课》。
    这门课强调 hands-on（动手实践）和真实工具体验。本课程借鉴它的实践取向，采用老师 Codex demo、学生本地模仿、打开网页验证的课堂节奏。
