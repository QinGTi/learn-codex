老师说前半句，你马上接后半句。不要想太久，像 AI 一样“根据前面的话猜下一句”。

例子：“今天放学以后，我最想……”

1

我打开冰箱

你会接什么？“拿饮料”？“找冰淇淋”？还是“发现没吃的”？

2

考试前一天

你会接什么？“复习到很晚”？“发现没带书”？上下文会影响答案。

3

如果我说错了

说明我不是查答案，而是在猜一个最可能的后续。

今天获得 4 张能力卡

大学课程会讲语言模型、训练、推理和评估。我们先抓住初一能用的直觉。

L

LLM

LLM（Large Language Model，大语言模型）：读过很多文字，能根据上下文生成回答的 AI 模型。

T

Token

Token（Token，模型处理文字的小片段）：模型看到的文字单位，可能是一个字、一个词或词的一部分。

Tr

Training

Training（Training，训练）：模型从大量例子中学习语言规律，就像看很多题后学会套路。

I

Inference

Inference（Inference，推理/生成）：模型根据你现在输入的话，生成它认为最可能有用的回答。

AI 看到的不是整篇作文

它会把文字切成小片段，再根据这些片段和上下文继续生成。

人看到的一句话

我今天想做一个英语单词复习计划。

模型可能看到的小片段

我 / 今天 / 想 / 做 / 一个 / 英语 / 单词 / 复习 / 计划

关键直觉

Token（Token，模型处理文字的小片段）越清楚，上下文越完整，LLM（Large Language Model，大语言模型）越不容易乱猜。

用 Codex 做接话小游戏

复制这段给 Codex。让它生成一个“前半句接后半句”的小游戏，并要求它解释为什么可能猜错。

复制这段 Prompt

你现在是我的 LLM 接话实验助手。请用初一学生能懂的话，帮我设计一个“根据前半句话猜后半句”的小游戏。 目标：让我理解 LLM（Large Language Model，大语言模型）不是答案库，而是在根据上下文生成最可能的回答。 要求： 1. 给我 5 个前半句。 2. 每个前半句给出 3 个可能的后半句。 3. 说明为什么不同后半句都可能成立。 4. 最后用 3 句话解释 Token（Token，模型处理文字的小片段）、Training（Training，训练）、Inference（Inference，推理/生成）。 限制： 1. 不要用大学生才懂的数学公式。 2. 每句话都要适合初一学生。 3. 不要说 LLM 一定正确，要提醒我它会猜错。

你要盯住 4 件事

  • 它有没有说清 LLM 不是答案库？
  • 它有没有给出多个可能答案？
  • 它有没有解释上下文为什么重要？
  • 它有没有承认自己可能猜错？

为什么 LLM 会出错？

不是因为它“故意骗人”，而是因为它经常在根据已知信息生成最可能的回答。

1

信息太少

上下文不够，它只能猜。

2

题目模糊

你没说清楚，它会自己补脑。

3

像真不等于真

句子顺，不代表事实对。

4

需要检查

重要答案要验证来源和逻辑。
